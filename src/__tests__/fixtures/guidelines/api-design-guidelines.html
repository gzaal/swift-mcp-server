Swift API Design Guidelines

Fundamentals

Clarity at the point of use is your most important goal. Code is read far more often than it is written.

Naming

Promote clear usage. Name variables, parameters, and associated types according to their roles.

Prefer method and function names that make use sites form grammatical English phrases.

Conventions

Follow case conventions. Names of types and protocols are UpperCamelCase. Everything else is lowerCamelCase.

Acronyms should be uniformly upper or lower case. Use HTTPRequest, not HttpRequest.

Special Instructions

Document the complexity of any computed property that is not O(1).
